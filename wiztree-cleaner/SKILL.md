## If no CSV provided

Ask for the export file path. Or if WizTree is installed on Windows, generate one via PowerShell (must use `-Verb RunAs` — see pitfall below):

```bash
powershell.exe -NoProfile -Command "Start-Process -FilePath 'D:\Windows\WizTree\WizTree64.exe' -ArgumentList 'D:\','/export=D:\exports\wt.csv','/exportfiles=1','/exportfolders=1','/exportalldates=1','/exportsplitfilename=1' -Verb RunAs -Wait"
```

`/exportalldates=1` adds last-accessed/created dates and `/exportsplitfilename=1` adds a separate extension column (both already in the command above) — adjust the drive letter and export path as needed.

### ⚠️ Pitfall: WizTree hangs from CLI — use PowerShell `-Verb RunAs`

WizTree has `actRunAsAdmin_Checked=TRUE` in its INI (`WizTree3.ini`, same install dir). This triggers a UAC elevation dialog that **blocks CLI execution silently** — no output, no CSV, the process just hangs forever.

**What does NOT work (confirmed in practice):**
1. ~~Using 32-bit `WizTree.exe` instead of `WizTree64.exe`~~ — WizTree.exe **launches WizTree64.exe internally**, so the hang still happens.
2. ~~Setting `actRunAsAdmin_Checked=FALSE` in the INI before launching~~ — WizTree64.exe **still hangs** even with admin disabled.
3. ~~MSYS bash direct invocation or `cmd.exe /c`~~ — MSYS bash mangles the Windows backslash arguments; `cmd.exe` wrappers exit without producing output.

**What DOES work** — the PowerShell `Start-Process -Verb RunAs` command shown above. Key details:
- **`-Verb RunAs`** is the critical flag — triggers the UAC elevation dialog WizTree expects.
- **`-Wait`** blocks until the export completes (~15–60s depending on drive size).
- Use **PowerShell** (not MSYS bash) — MSYS bash mangles backslash path arguments.

## Parse the CSV

Full-drive scans produce large CSVs (150–700 MB, 400K–2M rows). **Do not read the entire file as text** — use `execute_code` or `terminal` with Python.

### ⚠️ Pitfall: Line 0 is a donation message, NOT the header

WizTree CSVs start with a non-CSV line like:
```
Generated by WizTree 4.31 22/7/2026 8:43:00 AM (You can hide this message by making a donation)
```
The actual CSV header is on **line 1**. Using `csv.DictReader` without skipping this line causes all columns to be off by one, resulting in empty paths and zero sizes.

### ⚠️ Pitfall: pandas may not be available in the agent venv

The Hermes `execute_code` venv often lacks pandas. Use the **pure Python `csv` module** as a fallback — it handles 2M-row files fine and avoids dependency issues:

```python
import csv

csv_path = r"path\to\wt.csv"

with open(csv_path, 'r', encoding='utf-8-sig', errors='replace') as f:
    next(f)  # ← skip WizTree donation message line 0
    reader = csv.DictReader(f)
    for row in reader:
        path = row['File Name'].strip().strip('"')
        try:
            size = int(float(row.get('Size', 0) or 0))
        except (ValueError, TypeError):
            continue
        # Detect folders vs files — see below
        ...
```

### ⚠️ Pitfall: Folder vs file detection

**Do NOT use the `FILENAME` column to distinguish folders from files.** With `/exportsplitfilename=1`, both folders and files get a FILENAME value. Instead:

- **Folders** have paths ending with `\` (e.g., `C:\Windows\Temp\`)
- **Files** have paths NOT ending with `\` (e.g., `C:\pagefile.sys`)

```python
if path.endswith('\\'):
    # Folder row — Size = total size of all contents
    folders.append({'path': path, 'size': size, ...})
else:
    # Individual file row
    files.append({'path': path, 'size': size, ...})
```

Also skip the drive root row (path == `C:\`) — its Size is the drive capacity, not a cleanup target. Save `DRIVECAPACITY`, `FREESPACE`, `USEDSPACE` from that row for the report header.

### Column layouts

- **Basic:** `File Name, Size, Allocated, Modified, Attributes, Files, Folders`
- **Advanced** (`+ /exportalldates=1 /exportsplitfilename=1`): adds `LASTACCESSDATE, CREATEDDATE, ROOT, FOLDERNAME, FILENAME, FILEEXT, DRIVECAPACITY, FREESPACE, USEDSPACE, RESERVEDSPACE`

Key columns: `File Name` (full path), `Size` (bytes), `Modified` (datetime).

Folder rows have `Size` = total size of all contents — these are the primary targets for cleanup suggestions.

## Analyze

1. **Filter folders** (paths ending with `\`) sorted by `Size` descending — the top 50 are the space hogs that matter.
2. **Filter files** (paths NOT ending with `\`) sorted by `Size` descending for individual large files.
3. **Deduplicate parent-child overlap** (see below).
4. Match each entry against the pattern tables below.

### ⚠️ Pitfall: Parent-child folder double-counting

WizTree exports every folder AND its parent. `C:\Windows\` (47 GB) contains `C:\Windows\WinSxS\` (18 GB) which contains `C:\Windows\WinSxS\Backup\` (400 MB). If all three appear in the report, the reclaimable total is inflated.

**Dedup step:** After categorizing, sort items by path length ascending. For each item, check if any other item with the **same category** starts with this path — if so, skip it (the child is more specific and should be shown instead). This prevents the same category from appearing at multiple nesting levels:

```python
deduped = []
for item in sorted(items, key=lambda x: len(x['path'])):
    is_parent = False
    for other in items:
        if other['path'] != item['path'] and other['path'].startswith(item['path']) and other['category'] == item['category']:
            is_parent = True
            break
    if not is_parent:
        deduped.append(item)
```

Apply a **minimum size threshold** (100 MB for folders, 200 MB for individual files) before categorizing to keep the report focused.

## Categorize

### Safe to delete (✅)

| Pattern in `File Name` path | Category | Note |
|---|---|---|
| `\Temp\`, `\tmp\`, `*.tmp` | Temp files | Safe |
| `\AppData\Local\Google\Chrome\...\Cache`, `\...\Mozilla\Firefox\...\cache2`, `\...\Microsoft\Edge\...\Cache` | Browser cache | Regenerable |
| `\Windows\SoftwareDistribution\Download\` | Windows Update cache | Stop `wuauserv` service first |
| `\$Recycle.Bin\` | Recycle Bin | Ask before emptying |
| `*.log`, `*.evtx` (check `Modified` age) | Logs | Safe if old (>30 days) |
| `*.dmp`, `\Windows\Minidump\` | Crash dumps | Safe |
| `thumbcache_*.db`, `iconcache*.db` | Thumbnail cache | Regenerable |
| `\npm-cache\`, `\AppData\Local\npm-cache\` | npm cache | `npm cache clean --force` |
| `\pip\cache\`, `\AppData\Local\pip\Cache\` | pip cache | `pip cache purge` |
| `\node_modules\` (anywhere, incl. `\.cache\`, `\npm\`) | Dev dependencies | Safe if project inactive — run `npm prune` or delete folder |
| `\__pycache__\`, `*.pyc` | Python bytecode | Regenerable |
| `\bin\Debug\`, `\bin\Release\`, `\obj\` | Build output | Regenerable from source |
| `\.gradle\`, `\.m2\` | Gradle/Maven cache | Safe to clear |
| `\Downloads\*.exe`, `*.msi`, `*.iso` | Old installers | User discretion — check date |
| `\AppData\Local\CrashDumps\` | App crash dumps | Safe |
| `\Windows\Prefetch\` | Prefetch | Safe — Windows rebuilds it |
| `\AppData\Local\uv\cache\` | uv (Python) cache | `uv cache clean` |
| `\.cache\` (user home) | General cache dir | Regenerable — safe to clear |
| `\$WinREAgent\` | WinRE Agent temp | Safe to delete after Windows Update |
| `\.affinity\...\autosave\` | Affinity autosave | Delete old autosaves |
| `\Windows\Temp\*.etl`, `\Windows\SystemTemp\*.etl` | ETL trace logs | Safe — temp diagnostics |
| `\.codex\` (duplicate `codex.exe` copies) | Codex CLI copies | Keep one, delete rest |
| `claude.exe.old.*` | Old Claude CLI binary | Delete |

### Caution — don't delete directly (⚠️)

| Path | Correct action |
|---|---|
| `\Windows\WinSxS\` | `DISM /Online /Cleanup-Image /StartComponentCleanup` |
| `hiberfil.sys` | `powercfg /h off` to disable hibernation |
| `pagefile.sys` | System-managed — leave alone |
| `\System Volume Information\` | System Restore — never delete |
| `\Windows\Installer\` | Needed for repairs/uninstalls — use PatchCleaner if orphaned |
| `\Program Files*\` | Uninstall properly via Control Panel |
| `\Windows\System32\` | Never touch |
| `*.vhdx`, `*.vhd` (VM/WSL disks) | Review — VM disk, may be in use |
| `\Claude_pzs8...\...\vm_bundles\` | Claude desktop VM | Review — active VM data |
| `*.gguf`, `*.bin` (ML models) | ML model files | Review if still needed |

## Output

Present a prioritized table sorted by `Size` descending:

```
## Disk Cleanup — <drive> — reclaimable: XX GB

| # | Category | Path | Size | Safe? | Action |
|---|---|---|---|---|---|
| 1 | Temp files | C:\Windows\Temp\ | 3.2 GB | ✅ | Delete contents |
| 2 | Browser cache | ...\Chrome\...\Cache\ | 1.8 GB | ✅ | Delete contents |
| 3 | Dev artifacts | D:\Projects\app\node_modules\ | 1.1 GB | ✅ | Delete or `npm prune` |
| 4 | WinSxS | C:\Windows\WinSxS\ | 12 GB | ⚠️ | Run DISM cleanup |
```

End with: total reclaimable space split into ✅ Safe / ⚠️ Caution, plus copy-paste commands where applicable.

**Completion criterion:** every entry over 100 MB is categorized or flagged "review manually"; total reclaimable stated; safe items clearly separated from caution items.
